56 /*
57  * cdxCDynamicControlsManager
58  * ==========================
59  * Makes any CWnd derived class capable of dynamic control resizing
60  * and repositioning.
61  * Moreover, it can set a window's max/min tracking size (the size
62  * the user can change) and add a nice sizing icon to the windows
63  * lower right corner (if the window does not have one - as dialogs).
64  *
65  * To make any CWnd derived capable of automatically displaying
66  * its controls, you embed a member of this class in your window
67  * (or you derive your class from both this class and your window
68  * base class - that depends on how you want to use the member
69  * functions of this class).
70  *
71  * Then, the following functions must be called
72  *
73  * DoInitWindow() - Must be called after the window became
74  * valid (i.e. CWnd::m_hWnd is non-zero)
75  * and has its initial (minimum) size.
76  * DoOnSize() - by the OnSize() message handler
77  * DoOnGetMinMaxInfo() - by the OnGetMinMaxInfo() message handler
78  * DoDestroyWindow() - by DestroyWindow().
79  *
80  * See cdxCSizingDialog.h for an example.
81  *
82  * NOTE:
83  * Unfortunately, we cannot derive this class from CObject, because
84  * those macros DECLARE_xxx are too lame to handle multipile derived
85  * classes if both classes have been derived from the same base-class
86  * CObject.
87  */
